Core & Bean

@SpringBootApplication Khai báo class chính, tự động bật @Configuration + @ComponentScan + @EnableAutoConfiguration
@Component Đánh dấu class là Spring Bean (generic)
@Bean Khai báo method trả về bean trong @Configuration class
@Configuration Class chứa cấu hình, thường dùng cùng @Bean
@Autowired Tự động inject bean (nên dùng constructor injection thay thế)
@Qualifier("name") Chỉ định bean cụ thể khi có nhiều bean cùng kiểu
@Primary Ưu tiên bean này khi có nhiều bean cùng kiểu
@Scope("prototype") Tạo bean mới mỗi lần inject (mặc định là singleton)
@Lazy Khởi tạo bean khi cần, không khởi tạo lúc start app
🌐

Controller & Web

@RestController = @Controller + @ResponseBody, trả JSON/XML
@Controller Trả về view (Thymeleaf, JSP...)
@RequestMapping Map URL cho class hoặc method. VD: ("/api/tickets")
HTTP Methods
@GetMapping Lấy dữ liệu — GET /api/tickets
@PostMapping Tạo mới — POST /api/tickets
@PutMapping Cập nhật toàn bộ — PUT /api/tickets/1
@PatchMapping Cập nhật một phần — PATCH /api/tickets/1
@DeleteMapping Xóa — DELETE /api/tickets/1
Tham số
@PathVariable Lấy từ URL — /tickets/{id} → Long id
@RequestParam Query string — ?page=1&size=10
@RequestBody Parse JSON body thành object DTO
@RequestHeader Lấy giá trị từ HTTP header. VD: Authorization
@ResponseStatus Đặt HTTP status code mặc định. VD: (HttpStatus.CREATED)
@CrossOrigin Cho phép CORS trên từng controller/method
⚙️

Service & Stereotype

@Service Đánh dấu lớp business logic (extends @Component)
@Repository Đánh dấu lớp data access, tự dịch exception thành Spring DataAccessException
@Transactional Bọc method trong transaction DB. Tự rollback nếu có exception
@Transactional
(readOnly=true) Tối ưu cho query chỉ đọc, không ghi
🗄️

JPA & Entity

@Entity Ánh xạ class với bảng trong database
@Table(name="") Chỉ định tên bảng cụ thể. VD: @Table(name="tickets")
@Id Đánh dấu field là primary key
@GeneratedValue Tự tăng ID. VD: (strategy = GenerationType.IDENTITY)
@Column Tùy chỉnh cột — name, nullable, unique, length
@Enumerated Lưu enum vào DB. Dùng EnumType.STRING cho dễ đọc
@CreationTimestamp Tự ghi thời gian tạo (Hibernate)
@UpdateTimestamp Tự cập nhật thời gian sửa (Hibernate)
Quan hệ
@OneToOne 1 User ↔ 1 Profile
@OneToMany 1 User → nhiều Ticket
@ManyToOne Nhiều Ticket → 1 User
@ManyToMany Nhiều User ↔ nhiều Role
@JoinColumn Chỉ định foreign key. VD: name="user_id"
@JoinTable Chỉ định bảng trung gian cho @ManyToMany
✅

Validation

@Valid Kích hoạt validation cho @RequestBody object
@Validated Tương tự @Valid, hỗ trợ thêm group validation
Bean Validation (jakarta.validation)
@NotNull Không được null
@NotBlank Không được null, rỗng, hoặc chỉ có khoảng trắng
@NotEmpty Không được null hoặc rỗng (cho String, List...)
@Size(min,max) Giới hạn độ dài. VD: @Size(min=3, max=50)
@Min / @Max Giới hạn giá trị số
@Email Kiểm tra định dạng email hợp lệ
@Pattern(regexp) Kiểm tra theo regex. VD: số điện thoại
@Positive Số phải > 0
@Future / @Past Ngày phải trong tương lai / quá khứ
🚨

Exception Handling

@ControllerAdvice Xử lý exception toàn cục cho tất cả controller
@RestControllerAdvice = @ControllerAdvice + @ResponseBody, trả JSON
@ExceptionHandler Xử lý exception cụ thể. VD: (ResourceNotFoundException.class)
@ResponseStatus Đặt HTTP status cho exception. VD: (HttpStatus.NOT_FOUND)

Security

@EnableWebSecurity Bật Spring Security cho ứng dụng
@EnableMethodSecurity Cho phép dùng annotation bảo mật trên method
@PreAuthorize Kiểm tra quyền trước khi vào method. VD: ("hasRole('ADMIN')")
@PostAuthorize Kiểm tra quyền sau khi method chạy xong
@Secured Giới hạn role được phép. VD: ({"ROLE_ADMIN", "ROLE_USER"})
@AuthenticationPrincipal Inject thông tin user đang đăng nhập vào method
⚙️

Config & Properties

@Value Inject giá trị từ application.properties. VD: ("${jwt.secret}")
@ConfigurationProperties Map nhiều property vào một class. VD: (prefix="app.jwt")
@EnableConfigurationProperties Kích hoạt class @ConfigurationProperties
@Profile Bean chỉ load ở profile cụ thể. VD: ("dev") hay ("prod")
@ConditionalOnProperty Tạo bean khi property có giá trị nhất định
@PropertySource Load file properties tùy chỉnh vào context

Lombok (hay dùng)

@Data = @Getter + @Setter + @ToString + @EqualsAndHashCode
@Getter / @Setter Tự sinh getter/setter cho field hoặc cả class
@NoArgsConstructor Tạo constructor không tham số
@AllArgsConstructor Tạo constructor với tất cả field
@RequiredArgsConstructor Constructor cho các field final — dùng thay @Autowired
@Builder Tạo builder pattern. VD: User.builder().name("Nam").build()
@Slf4j Tự tạo biến log để ghi log. VD: log.info("message")

Testing

@SpringBootTest Load toàn bộ Application Context cho integration test
@WebMvcTest Chỉ load Controller layer, không cần DB
@DataJpaTest Chỉ load JPA layer, dùng H2 in-memory DB
@MockBean Tạo mock bean thay thế bean thật trong context
@Mock Tạo mock object (Mockito), dùng trong unit test
@InjectMocks Inject các @Mock vào class đang test
@Test Đánh dấu method là test case (JUnit 5)

Validation — Áp dụng thực tế

1. Thêm dependency vào pom.xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
2. Thêm annotation vào Record/DTO
public record RegisterRequest(

    @NotBlank(message = "Họ tên không được rỗng")
    @Size(min = 2, max = 50, message = "Họ tên từ 2-50 ký tự")
    String fullname,

    @NotNull(message = "Ngày sinh không được rỗng")
    @Past(message = "Ngày sinh phải là ngày trong quá khứ")
    LocalDate dob,

    @NotBlank(message = "Email không được rỗng")
    @Email(message = "Email không đúng định dạng")
    String email,

    @NotBlank(message = "Mật khẩu không được rỗng")
    @Size(min = 6, max = 20, message = "Mật khẩu từ 6-20 ký tự")
    String password

) {}
3. Thêm @Valid vào Controller
@PostMapping("/register")
public ResponseEntity<AuthResponse> register(@Valid @RequestBody RegisterRequest request) {
    // Nếu thiếu @Valid → annotation bị bỏ qua hoàn toàn!
    return ResponseEntity.ok(authService.register(request));
}
Phân biệt @NotNull / @NotEmpty / @NotBlank
// @NotNull  — chỉ kiểm tra null
null  → ❌   ""    → ✅   "   " → ✅   "abc" → ✅

// @NotEmpty — kiểm tra null + rỗng
null  → ❌   ""    → ❌   "   " → ✅   "abc" → ✅

// @NotBlank — kiểm tra null + rỗng + khoảng trắng ← dùng cho String!
null  → ❌   ""    → ❌   "   " → ❌   "abc" → ✅

GlobalExceptionHandler — Xử lý lỗi tập trung

@RestControllerAdvice  // Xử lý exception cho toàn bộ Controller
public class GlobalExceptionHandler {

    // Validation lỗi → 400 + danh sách field lỗi
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Map<String, String>> handleValidation(MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors()
          .forEach(e -> errors.put(e.getField(), e.getDefaultMessage()));
        return ResponseEntity.status(400).body(errors);
    }

    // Sai email/password → 401
    @ExceptionHandler(BadCredentialsException.class)
    public ResponseEntity<String> handleBadCredentials() {
        return ResponseEntity.status(401).body("Sai email hoặc mật khẩu");
    }

    // Tài khoản bị khóa → 403
    @ExceptionHandler(LockedException.class)
    public ResponseEntity<String> handleLocked() {
        return ResponseEntity.status(403).body("Tài khoản đã bị khóa");
    }

    // Email đã tồn tại, role không tìm thấy... → 400
    @ExceptionHandler(RuntimeException.class)
    public ResponseEntity<String> handleRuntime(RuntimeException ex) {
        return ResponseEntity.status(400).body(ex.getMessage());
    }
}

// Response khi validation lỗi:
// {
//   "email": "Email không đúng định dạng",
//   "password": "Mật khẩu từ 6-20 ký tự"
// }

// ✅ Entity — dùng @Getter + @NoArgsConstructor
@Entity
@Getter                    // tự tạo getter cho tất cả field
@NoArgsConstructor         // JPA cần constructor không tham số
public class User implements UserDetails {
    @Id private Long id;
    private String email;
    // KHÔNG dùng @Data cho Entity — gây lỗi với lazy loading!
}

// ✅ Service/Controller — dùng @RequiredArgsConstructor thay constructor thủ công
@Service
@RequiredArgsConstructor   // tự tạo constructor cho các field final

    private final UserRepository userRepository;    // ← tự inject

    // Không cần viết constructor nữa!
}

Đặt tên URL

✅ Đúng
danh từ số nhiều /api/tickets  /api/users  /api/comments
kebab-case /api/ticket-types  /api/user-roles
nested resource /api/tickets/{id}/comments — comment thuộc ticket
versioning /api/v1/tickets — nên có version từ đầu
❌ Sai
động từ trong URL /api/getTickets  /api/createUser  /api/deleteTicket
camelCase / snake_case /api/ticketTypes  /api/ticket_types
📋

HTTP Method + Status Code

GET /tickets → 200 OK + danh sách
/tickets/1 → 200 OK hoặc 404 Not Found
POST /tickets → 201 Created + object vừa tạo
PUT /tickets/1 → 200 OK + object đã cập nhật (toàn bộ)
PATCH /tickets/1 → 200 OK + object đã cập nhật (một phần)
DELETE /tickets/1 → 204 No Content (không trả body)
400 Bad Request — sai dữ liệu đầu vào, validation lỗi
401 Unauthorized — chưa đăng nhập / token hết hạn
403 Forbidden — đã login nhưng không có quyền
404 Not Found — không tìm thấy resource
409 Conflict — trùng dữ liệu (email đã tồn tại...)
500 Internal Server Error — lỗi server không xử lý được

Controller Pattern chuẩn — Ticket API

@RestController
@RequestMapping("/api/v1/tickets")
@RequiredArgsConstructor
public class TicketController {

    private final TicketService ticketService;

    // GET /api/v1/tickets?page=0&size=10&status=OPEN
    @GetMapping
    public ResponseEntity<ApiResponse<Page<TicketResponse>>> getAll(
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size,
            @RequestParam(required = false) TicketStatus status) {
        return ResponseEntity.ok(ticketService.getAll(page, size, status));
    }

    // GET /api/v1/tickets/1
    @GetMapping("/{id}")
    public ResponseEntity<ApiResponse<TicketResponse>> getById(@PathVariable Long id) {
        return ResponseEntity.ok(ticketService.getById(id));
    }

    // POST /api/v1/tickets
    @PostMapping
    public ResponseEntity<ApiResponse<TicketResponse>> create(@RequestBody @Valid TicketRequest req) {
        return ResponseEntity.status(201).body(ticketService.create(req));
    }

    // PATCH /api/v1/tickets/1
    @PatchMapping("/{id}")
    public ResponseEntity<ApiResponse<TicketResponse>> update(@PathVariable Long id,
            @RequestBody @Valid TicketRequest req) {
        return ResponseEntity.ok(ticketService.update(id, req));
    }

    // DELETE /api/v1/tickets/1
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        ticketService.delete(id);
        return ResponseEntity.noContent().build(); // 204
    }

    // GET /api/v1/tickets/1/comments
    @GetMapping("/{id}/comments")
    public ResponseEntity<ApiResponse<List<CommentResponse>>> getComments(@PathVariable Long id) {
        return ResponseEntity.ok(ticketService.getComments(id));
    }
}
